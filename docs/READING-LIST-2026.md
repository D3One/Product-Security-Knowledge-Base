# Reading List 2026

A curated **2026 bookshelf** for people working in and around **Product Security**.

The list mixes:

- foundational books that still hold up;
- practical books for current engineering work;
- cloud-native and API-focused titles;
- leadership books that help turn technical depth into program-level impact.

> The point of this list is not novelty for novelty’s sake. It is to give readers a **useful, durable, high-signal reading path** as of 2026.

## Threat Modeling, Secure by Design, and Foundations

- [Threat Modeling: Designing for Security](https://www.amazon.com/s?k=Threat+Modeling%3A+Designing+for+Security+Adam+Shostack) — **Adam Shostack**. The modern classic for practical threat modeling in software and systems development.
- [Threats: What Every Engineer Should Learn from Star Wars](https://www.amazon.com/s?k=Threats%3A+What+Every+Engineer+Should+Learn+from+Star+Wars+Adam+Shostack) — **Adam Shostack**. A more approachable follow-up that helps engineers build security intuition and judgment.
- [Secure by Design](https://www.amazon.com/s?k=Secure+by+Design+Dan+Bergh+Johnsson%2C+Daniel+Deogun%2C+Daniel+Sawano) — **Dan Bergh Johnsson, Daniel Deogun, Daniel Sawano**. Patterns and design habits for building security into software architecture from the beginning.
- [Security Engineering](https://www.amazon.com/s?k=Security+Engineering+Ross+Anderson) — **Ross Anderson**. A deep systems-level foundation for security architecture, controls, and dependable system design.
- [Alice and Bob Learn Application Security](https://www.amazon.com/s?k=Alice+and+Bob+Learn+Application+Security+Tanya+Janca) — **Tanya Janca**. A broad and approachable Application Security book with strong coverage of modern development realities.
- [Agile Application Security](https://www.amazon.com/s?k=Agile+Application+Security+Laura+Bell%2C+Michael+Brunton-Spall%2C+Rich+Smith) — **Laura Bell, Michael Brunton-Spall, Rich Smith**. Excellent for embedding security into agile delivery, product teams, and CI/CD reality.

## Application Security, Web Security, Secure Coding, and Testing

- [Alice and Bob Learn Secure Coding](https://www.amazon.com/s?k=Alice+and+Bob+Learn+Secure+Coding+Tanya+Janca) — **Tanya Janca**. A developer-friendly secure coding guide that makes concrete defensive habits easier to operationalize.
- [Web Security for Developers](https://www.amazon.com/s?k=Web+Security+for+Developers+Malcolm+McDonald) — **Malcolm McDonald**. A practical defensive web security book written for people who actually ship web software.
- [The Tangled Web](https://www.amazon.com/s?k=The+Tangled+Web+Michal+Zalewski) — **Michal Zalewski**. Still one of the best books for understanding browser behavior and the weirdness of the web platform.
- [Web Application Defender's Cookbook](https://www.amazon.com/s?k=Web+Application+Defender%27s+Cookbook+Ryan+C.+Barnett%2C+Jeremiah+Grossman) — **Ryan C. Barnett, Jeremiah Grossman**. A defensive companion to offensive web knowledge, with concrete mitigation guidance.
- [Real-World Cryptography](https://www.amazon.com/s?k=Real-World+Cryptography+David+Wong) — **David Wong**. Useful when Product Security work touches auth, crypto decisions, token systems, and protocol literacy.
- [Fuzzing for Software Security Testing and Quality Assurance](https://www.amazon.com/s?k=Fuzzing+for+Software+Security+Testing+and+Quality+Assurance+Ari+Takanen%2C+Jared+DeMott%2C+Charles+Miller) — **Ari Takanen, Jared DeMott, Charles Miller**. A valuable pick for teams that want deeper testing maturity, especially around robustness and parser abuse.

## API Security, Identity, and Distributed Systems

- [API Security in Action](https://www.amazon.com/s?k=API+Security+in+Action+Neil+Madden) — **Neil Madden**. One of the most practical API security books for real teams building and protecting APIs.
- [Secure APIs](https://www.amazon.com/s?k=Secure+APIs+Jos%C3%A9+Haro+Peralta) — **José Haro Peralta**. Good design-first treatment of API security, including common risks and implementation tradeoffs.
- [OAuth 2 in Action](https://www.amazon.com/s?k=OAuth+2+in+Action+Justin+Richer%2C+Antonio+Sanso) — **Justin Richer, Antonio Sanso**. Strong grounding for teams that need to understand modern delegated auth beyond blog-post depth.
- [Microservices Security in Action](https://www.amazon.com/s?k=Microservices+Security+in+Action+Prabath+Siriwardena%2C+Nuwan+Dias) — **Prabath Siriwardena, Nuwan Dias**. A useful bridge across APIs, service identity, microservices, and Kubernetes-era architecture.

## Cloud Security, DevSecOps, Supply Chain, and Resilience

- [Practical Cloud Security](https://www.amazon.com/s?k=Practical+Cloud+Security+Chris+Dotson) — **Chris Dotson**. One of the clearest cloud security books for real architecture, IAM, asset management, and operations.
- [Securing DevOps](https://www.amazon.com/s?k=Securing+DevOps+Julien+Vehent) — **Julien Vehent**. A practical DevSecOps/cloud security book with a strong operational mindset.
- [Learning DevSecOps](https://www.amazon.com/s?k=Learning+DevSecOps+Steve+Suehring) — **Steve Suehring**. A newer process-and-tools guide that helps frame how DevSecOps works in practice.
- [Security Chaos Engineering](https://www.amazon.com/s?k=Security+Chaos+Engineering+Kelly+Shortridge%2C+Aaron+Rinehart) — **Kelly Shortridge, Aaron Rinehart**. Especially useful for mature programs focused on resilience, systemic thinking, and validation.

## Container and Kubernetes Security

- [Container Security](https://www.amazon.com/s?k=Container+Security+Liz+Rice) — **Liz Rice**. A foundational cloud-native security book that explains container risk, primitives, and defenses clearly.
- [Hacking Kubernetes](https://www.amazon.com/s?k=Hacking+Kubernetes+Andrew+Martin%2C+Michael+Hausenblas) — **Andrew Martin, Michael Hausenblas**. Threat-driven Kubernetes hardening with offensive context and practical defensive recommendations.
- [Kubernetes Security and Observability](https://www.amazon.com/s?k=Kubernetes+Security+and+Observability+Brendan+Creane%2C+Amit+Gupta%2C+and+contributors) — **Brendan Creane, Amit Gupta, and contributors**. Useful for teams that want to align runtime visibility with actual security control implementation.

## Leadership, Program Design, Metrics, and Career Growth

- [Security for Everyone](https://www.amazon.com/s?k=Security+for+Everyone+Laura+Bell+Main%2C+Erica+Anderson) — **Laura Bell Main, Erica Anderson**. A pragmatic security leadership book for smaller and growing organizations.
- [Security Metrics: A Beginner's Guide](https://www.amazon.com/s?k=Security+Metrics%3A+A+Beginner%27s+Guide+Caroline+Wong) — **Caroline Wong**. Still useful for leaders who need to frame metrics, targets, narratives, and stakeholder buy-in.
- [The Manager's Path](https://www.amazon.com/s?k=The+Manager%27s+Path+Camille+Fournier) — **Camille Fournier**. Not a security book, but excellent for technical leadership growth from engineer to manager.
- [An Elegant Puzzle](https://www.amazon.com/s?k=An+Elegant+Puzzle+Will+Larson) — **Will Larson**. Strong for engineering management systems thinking, especially in scaling organizations.
- [Staff Engineer](https://www.amazon.com/s?k=Staff+Engineer+Will+Larson) — **Will Larson**. Helpful for senior IC influence, leadership without line management, and architectural impact.
- [Engineering Management for the Rest of Us](https://www.amazon.com/s?k=Engineering+Management+for+the+Rest+of+Us+Sarah+Drasner) — **Sarah Drasner**. A modern, readable guide for managers building teams and operating models.
- [Cyber for Builders](https://www.amazon.com/s?k=Cyber+for+Builders+Ross+Haleliuk) — **Ross Haleliuk**. Best fit for people thinking about cybersecurity products, strategy, and the business side of security.
- [The Phoenix Project](https://www.amazon.com/s?k=The+Phoenix+Project+Gene+Kim%2C+Kevin+Behr%2C+George+Spafford) — **Gene Kim, Kevin Behr, George Spafford**. Not security-specific, but still valuable for understanding delivery flow, constraints, and DevOps culture.

## How to approach the list

- If you are early in Product Security, start with **Alice and Bob Learn Application Security**, **Threat Modeling**, **Practical Cloud Security**, and **Container Security**.
- If you already work in the field, use this list by **domain gap**: API, cloud, Kubernetes, metrics, leadership, or architecture.
- If you are moving toward manager / director scope, combine technical books with **Security Metrics**, **The Manager’s Path**, **An Elegant Puzzle**, and **Security Chaos Engineering**.
