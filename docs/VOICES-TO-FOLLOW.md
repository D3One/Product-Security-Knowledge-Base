# Voices to Follow

A curated list of people whose writing, speaking, building, and public educational work repeatedly add value around **Product Security and adjacent domains**.

> This page is **not** a universal ranking. It is a curated follow-list for practitioners who want more signal, stronger judgment, and better long-term learning inputs.

## Fifteen strong voices in and around Product Security

| Person | Why follow them |
|---|---|
| [Adam Shostack](https://shostack.org/) | Threat modeling, Secure by Design thinking, and security strategy for engineers and leaders. |
| [Tanya Janca](https://shehackspurple.ca/) | Application security education, secure coding, developer enablement, and accessible training. |
| [Laura Bell Main](https://www.laurabellmain.com/) | Secure development culture, pragmatic AppSec, and security leadership that works with real teams. |
| [Neil Madden](https://neilmadden.blog/about/) | API security, applied cryptography, OAuth, and practical identity-centric design. |
| [Josh Bressers](https://cloudsecurityalliance.org/profiles/josh-bressers) | Product security, OSS security, vulnerability management, and supply chain realism. |
| [Kelly Shortridge](https://kellyshortridge.com/) | Resilience, Security Chaos Engineering, product strategy, and modern defensive thinking. |
| [Jim Manico](https://manicode.com/) | Secure coding, AppSec education, OWASP leadership, and developer-first security practices. |
| [Caroline Wong](https://www.rsaconference.com/experts/caroline-wong) | Security metrics, executive communication, product security strategy, and security leadership. |
| [Liran Tal](https://lirantal.com/) | Developer security, open source security, AI security, and practical software risk reduction. |
| [Ross Haleliuk](https://ventureinsecurity.net/) | Cybersecurity product thinking, industry analysis, and leadership/business perspective for security builders. |
| [Marco Lancini](https://www.marcolancini.it/about/) | Cloud security, Kubernetes, cloud-native engineering, and technical leadership. |
| [Steven Lipner](https://www.stevelipner.org/about_me/default.html) | Foundational secure development leadership, Microsoft SDL history, and software assurance. |
| [Dan Guido](https://trailofbits.com/about/) | Offensive-informed product security, software assurance, AI-augmented security practice, and high-end research. |
| [Alyssa Miller](https://alyssasec.com/about) | Application security advocacy, leadership, and security strategy with a practitioner voice. |
| [Clint Gibler](https://tldrsec.com/authors/clint-gibler) | High-signal research curation, modern AppSec/Product Security thinking, and strong educational content. |

## Suggested way to use this list

- Follow a mix of **framework thinkers**, **hands-on engineers**, and **security leaders**.
- Avoid building your worldview from only one lane such as pure cloud, pure AppSec, or pure management.
- Revisit this page over time: the best people to follow are the ones who keep shipping insight, not just hot takes.
